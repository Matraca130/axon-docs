================================================================================
KEY DESIGN DECISIONS
================================================================================

DECISION 1: Agenda as PRIMARY, Month as SECONDARY
  Rationale: Month grid at 375px is unreadable with heatmap + events
  Implementation: Tab toggle allows both views; agenda is default

DECISION 2: Bottom Sheet for Exam Details
  Rationale: Natural swipe-to-dismiss; preserves calendar context; thumb-friendly
  Implementation: RDK Dialog + React touch handlers (no extra libraries)
  Contrast with: Centered modal (bad on 375px); full-screen overlay (overkill)

DECISION 3: Swipe Gestures (No Click Arrows)
  Rationale: Native mobile gesture; reduces UI clutter
  Implementation: useSwipe() hook (provided); 50px threshold; vertical priority
  Testing: Test on iPhone + Android (platforms differ)

DECISION 4: Heatmap + Indicators Layered
  Rationale: 48×48px cell can't fit text + color + dots if side-by-side
  Implementation: Heatmap = background color; dots/rings = position: absolute
  Example: Red cell + 2px red dot = "multiple exams today"

DECISION 5: Sticky Bottom Tab Bar (Not Top Nav)
  Rationale: Bottom is thumb-reachable; top wastes vertical space on short screens
  Implementation: 56px height + iOS safe-area-inset-bottom
  Content: pb-20 padding to avoid overlap

================================================================================
COLOR SYSTEM (HEATMAP + URGENCY)
================================================================================

Heatmap (calendar day cells):
  none   → bg-white (no events)
  study  → bg-blue-50 (study session)
  review → bg-teal-50 (repaso)
  exam   → bg-orange-50 (exam day)
  urgent → bg-red-50 (exam soon)

Dark mode equivalents provided (bg-[color]-900, text-[color]-100)

Urgency badges (exam cards):
  > 30 days   → gray (hay tiempo)
  15-30 days  → blue (empeza a planificar)
  7-14 days   → amber (intensificar)
  3-7 days    → orange (modo intensivo)
  < 3 days    → red (último repaso)

All verified for WCAG AA contrast (4.5:1) on light + dark backgrounds.

================================================================================
PERFORMANCE TARGETS
================================================================================

FCP (First Contentful Paint):  < 2.5s on slow 4G
INP (Interaction to Paint):    < 200ms
Memory usage:                  < 50 MB after 5 min
Swipe frame rate:              60 FPS (48+ acceptable on budget phones)

Optimizations provided:
  ✓ Virtual scrolling for agenda (render only visible events + 5 buffer)
  ✓ Lazy-load details panel (split route)
  ✓ Defer heatmap color fetch (render gray first, load real colors async)
  ✓ Debounce swipe events (prevent rapid API calls)

================================================================================
ACCESSIBILITY (WCAG 2.1 AA)
================================================================================

Touch targets:      ✓ 44×44px minimum (cells 48×48px, buttons 44px+)
Color contrast:     ✓ 4.5:1 (verified for all heatmap colors + dark mode)
Keyboard nav:       ✓ Tab through events, arrows navigate weeks, Enter opens
ARIA labels:        ✓ Aria-label on cards: "Examen de X en Y días"
Screen reader:      ✓ Announces exam details on focus
Reduced motion:     ✓ Respects prefers-reduced-motion (no swipe animations)
